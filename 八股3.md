# 42. 野指针和悬空指针的概念？

| 类型         | 定义                                 | 原因及危害                                                   |
| ------------ | ------------------------------------ | ------------------------------------------------------------ |
| **野指针**   | 指针变量指向一个**随机的地址**       | 通常是因为没有初始化指针变量。使用野指针可能会读取到不属于程序的内存区域，导致数据损坏或程序崩溃 |
| **悬空指针** | 指向了一块**已经被释放的内存的指针** | 如果在释放地址后没有将指针设置为`nullptr`，再次使用这个指针可能会发生未定义的行为，如访问已释放的内存。 |

​	**(1)野指针：**野指针则是**指向不确定内存区域的指针**。这通常是因为**指针变量未被初始化就开始使用**，或是**指向的内存已被释放**（比如用 `free` 或 `delete` 归还给系统）或**重新用于其他用途**后（比如同一个地址被新的变量占用），**该指针未被立即清零或重新赋值**，它仍然傻傻地保存着原来的地址值。此时，这个指针就成了“野针”——它指向的是一块“**已经不再属于你**”或者“**用途已变**”的内存。

野指针的危害在于它可能指向任何位置，对其的任何操作都可能导致不可预测的结果，如程序崩溃或数据损坏。

“野指针”被当作一个 **更宽泛的概念**——**任何指向无效（非法、不可预知）内存区域的指针**，都叫野指针。这样分类时：

- **未初始化的指针**（狭义野指针） → 指向“绝对随机”的地址。
- **已释放未置空的指针**（**悬空指针**） → 指向“曾经合法但当前已不合法”的地址。

​	**(2)悬空指针：**悬空指针是**指向一块已经被释放的内存的指针**。这通常发生在对象或内存块被删除后，指向该内存的指针没有被及时设为 `nullptr`。这种指针的危险在于，如果继续使用它，可能会操作一些已被操作系统回收或重新分配给其他程序的内存，从而导致数据损坏或程序异常。

- **防止错误：**某些处理器要求数据严格对齐，否则可能导致运行时错误。
- **兼容性：**正确的内存对齐保证了数据结构在不同计算机系统之间能够正确读取和写入，尤其是在涉及网络通信和文件存储的应用中。

## 什么是栈

在C++中，当一个函数被调用时，程序会通过**栈**(`stack`)来管理函数的执行流程。栈是一种**后进先出**的数据结构，函数的调用和返回都会涉及到栈的操作。

## 什么是栈帧

- **每个函数调用时，都会在栈上分配一端连续内存空间，称为栈帧（`stack frame`）**
- **栈帧用于存储 函数的局部变量、参数、返回地址 以及调用者的上下文信息（如调用者的栈帧基址）。**

栈是一段从高地址向低地址生长的连续内存。每次函数调用，都会在栈顶“切”出一块专属空间给该函数使用，这块空间就叫**栈帧**。它通常包含：

- **函数参数**（部分可能通过寄存器传递）
- **返回地址**（调用完下一条指令的地址）
- **保存的基址指针**（用于恢复上一个栈帧）
- **局部变量**
- **临时数据**（计算中间值）

其中关键的寄存器角色：

- **ESP**（栈指针）：始终指向栈顶（当前可用地址）
- **EBP**（基址指针）：指向当前栈帧的底部（固定锚点）

​	**函数调用的压栈过程表格**

| **步骤**           | **描述**                   |
| ------------------ | -------------------------- |
| **参数压栈**       | **从右到左将参数压入栈中** |
| **返回地址压栈**   | 将函数返回地址压入栈中     |
| **跳转到函数代码** | 程序跳转到函数代码执行     |
| **函数体执行**     | 在函数体中执行具体的代码   |
| **返回值处理**     | 将返回值存储在寄存器或栈中 |
| **栈清理和返回**   | 清理栈并返回到调用位置     |

![压栈](D:\Typora\typora_work\C++规划\压栈.png)

函数调用约定:

- `__cdecl`(C调用约定)
  - **参数压栈顺序**:从右到左
  - **返回地址压栈**:在参数之后
  - **栈清理**:由**调用者**清理栈
  - `int __cdecl add(int a,int b)`
- `__stdcall`(标准调用约定)
  - **参数压栈顺序**:从右到左
  - **返回地址压栈**:在参数之后
  - **栈清理**:由**被调用者**清理栈
  - Windows API 中常用

```c++
int fun1(int a,int b,int c){
    int sum = a + b + c;
    return sum;
}

int main(){
    int result = fun1(1,2,3);
    std::cout<<"result："<<result<<std::endl;
    return 0;
}
```

| **事件驱动编程** | 在图形用户界面（GUI）编程中，回调函数用于处理**按钮点击、鼠标移动**等事件。例如，点击按钮时调用一个特定的函数来处理该点击事件。 | GUI 编程中处理按钮点击、鼠标移动等事件回调。    |

- **回调函数**是一种委托机制，通过把函数的指针作为参数传递给另一个函数，从而在该函数内部调用指针指向的函数。换句话说，**回调函数是由其他代码在适当的时间点调用的函数**。

```c++
#include <iostream>

//定义回调函数类型
typedef void (*CallBackFunction)(int);


void myCallBackFunction(int num){
    std::cout<<"Callback Function,value:"<<num<<std::endl;
}

//一个接受回调函数作为参数的函数
//一个函数将一个指针作为参数,这个指针指向的是回调函数
void performAction(int value,CallBackFunction callback){//第二个形参为CallbackFuction指针的实体
    //在某些条件下调用回调函数
    if(value > 0){
        //当callback是一个类型为void(*)int的函数指针时,
        // callback(5);就是通过该指针调用它所指向的函数，并传递参数 5。
        //因为在C/C++中,函数指针可以直接像函数名那样使用
        callback(value);//相当于myCallBackFunction(value);
    }
}

int main(){
    int num = 5;
    performAction(num,myCallBackFunction);//第二个实参为函数名,表示函数地址
}
```



# 49. 内联函数和宏定义的区别

​	**表. 内联函数和宏定义对比表格**

| **特性**     | **内联函数**               | **宏定义**                 |
| ------------ | -------------------------- | -------------------------- |
| **类型检查** | 进行类型检查               | 不进行类型检查             |

内联函数和宏定义在性能优化上有相似之处，但**内联函数更安全、更灵活**。内联函数支持类型检查和调试，适用于短小的函数。宏定义通过预处理器进行文本替换，缺乏类型检查，容易导致难以发现的错误。

```c++
#include <iostream>
#define SQUARE(x) (x * x)//宏定义 正确的定义((x)*(x))
inline int square(int x){
    return x * x;
}

int main(){
    int a = 5;
    std::cout<<"Macro Square："<<SQUARE(a+1)<<std::endl;//输出11,a+1*a+1
    std::cout<<"Inline Function Square："<<square(a+1)<<std::endl;//输出36,(a+1)*(a+1)
    return 0;
}
```

# 51. 深拷贝浅拷贝区别

**浅拷贝**只复制对象的指针,不复制指针指向的内存。多个对象共享同一块内存资源。一个对象的修改会影响共享内存的所有对象。

**深拷贝**复制对象及其指针指向的所有内存内容。每个对象拥有自己独立的内存副本。一个对象的修改不会影响其他对象。

 [浅拷贝与深拷贝.md](..\c++_md\2面向对象编程\2引用以及拷贝构造和拷贝赋值\浅拷贝与深拷贝.md) 

深拷贝（拷贝指针变量指向的内存区域）

浅拷贝（只拷贝指针变量的值，不拷贝指针变量指向的内存区域）

**深拷贝使用情况**：含有指针类型的成员变量

**其他情况下**：直接使用默认拷贝构造和拷贝赋值就行

（**浅拷贝只要不涉及到指针都是没有问题的**）

## **含有指针变量时，自定义新的拷贝构造拷贝赋值**

```c++
#include<iostream>
using namespace std;
class A
{
private:
    double m_height;
    int* m_p_age;
    string m_name;

public:
   A(double height = 0, int age = 0, const string&name = "") 
   :m_height(height),m_p_age(new int(age)), m_name(name)
    {
        cout << "构造函数" << endl;
       	//m_p_age(new int(age)) 先为age在堆中申请一个int大小的内存地址空间，然后将age的地址给到m_p_age;
        // 即m_p_age 不再是一个野指针,而是合法的指向我们申请的堆地址int* m_p_age = &age;
        //堆区：是供程序员  手动管理  的动态内存区域，可以在程序运行时按需分配和释放。
    }
    ~A()
    {
        cout << "析构函数" << endl;
        delete m_p_age;
    }
    void print()
    {
        cout << "m_height = " << m_height << ", *m_p_age = " << *m_p_age 
        <<",m_name = " << m_name << endl;
    }
    A(const A& that)//深拷贝
    {
        cout<<"拷贝构造函数的深拷贝"<<endl; //A a2(a1);
        m_name = that.m_name;
        m_height = that.m_height;
        m_p_age = new int(*that.m_p_age);
        // that.m_p_age 是一个指向 int 的指针，表示它存储着一个 int 类型的值的内存地址。
        // 使用 * 运算符来解引用指针，访问该指针指向的整数值，也就是获取原对象 that 中的年龄值。
        //申请一块新内存，把原对象 age 的值复制进去，返回新内存的地址，然后赋给当前对象的 m_p_age。
        //new的使用:new int(初始值)
       		// - 它会在堆上动态分配一块足以存放 int 的内存，并用括号内的值（这里是 *that.m_p_age，即源对象的年龄值）来初始化这块内存。
    }
  
    // 返回值引用，避免多调用拷贝构造
    A& operator = (const A& that)//深拷贝
    {
```

# 53. 说一下零拷贝

零拷贝技术**在数据传输过程中避免了内存拷贝**，直接在不同模块（如用户空间和内核空间）之间传递数据，从而提高了系统性能，**减少了 CPU 和内存带宽的消耗**。常用于**高性能网络传输和文件 I/O 操作**，典型实现包括 `sendfile`、`mmap`、`splice` 和 `DMA` 技术。

## 1. 详细解释

**1）定义：**零拷贝技术使得数据在处理过程中不需要从一个缓冲区拷贝到另一个缓冲区，而是通过指针或引用传递。这减少了内存带宽的消耗和 CPU 的负载

**2）性能提升：**由于减少了数据在内存中的拷贝操作，提高了数据传输的速度。

**3）降低 CPU 负载：**避免了不必要的内存拷贝操作，减少了 CPU 的使用率。

**4）内存带宽利用率：**减少了内存总线的使用，提高了系统整体性能。

## 2. 使用场景

**1）高性能网络传输：**例如服务器需要快速传输大量数据时，零拷贝技术可以显著提高传输效率。

**2）文件 I/O 操作：**例如文件复制、文件发送等场景，使用零拷贝可以减少文件在用户空间和内核空间之间的拷贝。

## 3. 常见实现技术

**1）sendfile：**在 Linux 中，将文件数据直接从文件描述符发送到网络套接字。

**2）mmap：**内存映射文件，允许文件的内容直接映射到进程的地址空间。

**3）splice：**将数据在两个文件描述符之间移动，而无需将数据拷贝到用户空间。

**4）DMA（Direct Memory Access）：**允许设备直接访问主内存，减少 CPU 参与数据传输。
