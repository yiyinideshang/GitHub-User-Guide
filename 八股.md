# 1. C++ 三大特性

## 一、封装（Encapsulation）

- **概念**：将数据（**属性**）和操作数据的**方法**（函数）捆绑在一起，并隐藏内部实现细节，只暴露必要的接口。
- **实现**：使用 `class` 或 `struct`，通过 `public`、`protected`、`private` 控制访问权限。
- **作用**：
  - 数据保护，防止外部非法访问,**提高安全性**。
  - **降低模块间耦合**，**便于维护**。
  - 隐藏实现细节，方便重构。

**面试示例**（动物类）：

```cpp
class Animal {
private:
    std::string name;
    int age;
public:
    Animal(std::string n, int a) : name(n), age(a) {}
    virtual void speak() const = 0;
    virtual void info() const {
        std::cout << "Name: " << name << ", Age: " << age << std::endl;
    }
};
```
- `name` 和 `age` 私有，只能通过 `info()` 访问，外部无需关心内部存储方式。

## 二、继承（Inheritance）

### 1. 概念
- 派生类基于基类构建，自动获得基类成员，并可扩展或重定义功能。
- 目的：**代码复用**、**建立类型层次**、为**多态**提供基础。

### 2. 继承的内容
- **所有数据成员**（包括私有成员，但不可直接访问）。
- **普通成员函数**（包括静态成员函数，但构造函数、析构函数等除外）。

### 3. 不继承的内容
- 构造函数（拷贝构造函数、移动构造函数）
- 析构函数
- 拷贝赋值运算符、移动赋值运算符
- 友元关系

**为什么不继承？**
- **构造函数/析构函数**：每个类有独立的初始化/清理逻辑，派生类需先调用基类构造，再处理自己的成员；若继承则无法处理新增成员。
- **赋值运算符**：基类版本只能复制基类部分，会遗漏派生类成员，导致对象不完整。
- **友元**：友元关系不能传递，基类的友元无权访问派生类私有成员。

### 4. 实现方式（继承权限）
| 继承方式    | 基类成员在派生类中的访问级别                                 | 典型用途               |
| ----------- | ------------------------------------------------------------ | ---------------------- |
| `public`    | `public` → `public`<br>`protected` → `protected`<br>`private` → 不可访问 | **is‑a** 关系，最常用  |
| `protected` | `public` → `protected`<br>`protected` → `protected`<br>`private` → 不可访问 | 实现继承，限制对外暴露 |
| `private`   | 所有成员 → `private`（不可访问）                             | 组合的替代，接口隔离   |

### 5. 作用
- 减少重复代码，提高复用性。
- 建立类层次结构（如 `Dog`、`Cat` 继承 `Animal`）。
- 为运行时多态提供支持。

**面试示例**（狗和猫）：
```cpp
class Dog : public Animal {
public:
    Dog(std::string n, int a) : Animal(n, a) {}
    void speak() const override { std::cout << "Woof!" << std::endl; }
};
class Cat : public Animal {
public:
    Cat(std::string n, int a) : Animal(n, a) {}
    void speak() const override { std::cout << "Meow!" << std::endl; }
};
```
- 创建 `Dog` 对象时，先调用 `Animal` 构造函数，再调用 `Dog` 构造函数，确保基类部分先初始化。

## 三、多态（Polymorphism）

### 1. 概念
- 同一操作作用于不同对象时，表现出不同的行为。
- **编译时多态**：函数重载、模板（静态绑定）。
- **运行时多态**：虚函数（动态绑定）。

### 2. 实现
- 基类声明 `virtual` 函数，派生类**覆盖**（`override`）。
- 通过**基类指针或引用**调用虚函数，运行时根据实际对象类型执行相应版本。

### 3. 作用
- 增强代码灵活性和可扩展性。
- 实现“一个接口，多种方法”，符合**开闭原则**。

**面试示例**（动物园）：

```cpp
std::vector<Animal*> zoo;
zoo.push_back(new Dog("Buddy", 5));
zoo.push_back(new Cat("Whiskers", 3));

for (auto animal : zoo) {
    animal->speak();   // 输出 Woof! 和 Meow!
}
```

**补充细节**（加分项）：
- **虚函数表（vtable）**：每个含虚函数的类有一个虚函数表，对象中隐藏一个指向该表的指针，运行时动态查找函数地址。
- **纯虚函数与抽象类**：`virtual void speak() const = 0;` 使 `Animal` 成为抽象类，不能实例化，更符合设计意图。

# 2. C++ 中 `struct` 和 `class` 的区别

## 1. 对比表格

| 特性                | `struct`                                                     | `class`                                              |
| :------------------ | :----------------------------------------------------------- | :--------------------------------------------------- |
| **默认访问修饰符**  | `public`                                                     | `private`                                            |
| **默认继承方式**    | `public` 继承                                                | `private` 继承                                       |
| **用途风格**        | 传统用于**简单数据结构**（POD），成员以数据为主              | 用于**数据封装和行为**的复杂对象                     |
| **功能支持**        | 支持成员函数、构造/析构、继承、多态、模板等**所有C++特性**   | 同样支持所有C++特性                                  |
| **与C语言的兼容性** | 与C语言的 `struct` **语法兼容**（但C++的 `struct` 可以包含成员函数等，不能直接用于C编译器） | 不与C语言结构兼容 是C++独有，无法直接在C语言中使用。 |

> **核心要点**：在C++中，`struct` 和 `class` 在语法和功能上**完全一致**，唯一的区别是**默认访问权限**和**默认继承方式**不同。其他所有特性（如成员函数、继承、多态、模板等）两者都支持。

## 面试回答策略

**简洁版**（适合快速回答）：

> “在C++中，`struct` 和 `class` 在功能上完全等价，唯一的区别是**默认访问权限**和**默认继承方式**：`struct` 默认 `public`，`class` 默认 `private`。通常 `struct` 用于简单数据结构，`class` 用于复杂**封装**对象。”

**详细版**（如果面试官追问）：

> “除了默认访问权限和继承方式不同外，两者都支持成员函数、构造/析构、继承、多态、模板等所有C++特性。`struct` 在语法上与C语言结构兼容，常用于跨语言的数据交换。选择 `struct` 还是 `class` 更多是代码风格和设计意图的体现：当对象以数据为主且对外完全公开时用 `struct`；当需要封装和行为时用 `class`。”

# 3. 结构体 struct 和联合体 union 的区别

| 特性         | struct                                                       | union                                                        |
| :----------- | :----------------------------------------------------------- | :----------------------------------------------------------- |
| **内存分配** | 每个成员拥有独立的内存空间，按声明顺序依次分配               | 所有成员共享同一块内存区域，同一时刻只能存储一个成员         |
| **大小**     | **至少为所有成员大小之和**，但受内存对齐影响，实际大小往往是**最大成员对齐值的整数倍** | 大小为**最大成员的大小**（也需对齐到最大成员的对齐要求）     |
| **内存对齐** | 编译器会在成员之间插入填充字节，保证每个成员按自身对齐要求存储 | 整体对齐到最大成员的对齐要求，内部无成员间填充               |
| **使用场景** | 存储多个**同时存在**的、相关的数据                           | 存储**互斥使用**的多种数据类型，用于节省内存                 |
| **访问**     | 可以同时访问所有成员                                         | **只能访问当前最后写入的成员**，读取其他成员可能导致未定义行为 |
| **数据类型** | 常用于表示一个具有多种属性的对象                             | 常用于变体类型、协议解析、节省内存的场合                     |

> “结构体和联合体的主要区别在于内存布局：结构体的每个成员都有独立空间，大小受成员之和与内存对齐共同决定，通常是最大成员对齐值的整数倍；而联合体的所有成员共享同一块内存，大小等于最大成员的大小，也需对齐到最大成员的对齐要求。结构体适合存储多个同时存在的属性，联合体适合节省内存的场景，比如协议解析或变体类型。”

### 例1：最大成员大小刚好是对齐值的整数倍

```cpp
union U1 {
    char c;   // 1 字节，对齐要求 1
    int i;    // 4 字节，对齐要求 4
};
```

- 最大成员大小 = 4 字节
- 最大对齐要求 = 4
- 4 已经是 4 的倍数 → 大小 = 4 字节

------

### 例2：最大成员大小不是对齐值的整数倍

```cpp
union U2 {
    char arr[5];  // 5 字节，对齐要求 1
    int i;        // 4 字节，对齐要求 4
};
```

- 最大成员大小 = 5 字节（arr 的大小）
- 最大对齐要求 = max(1,4) = 4
- 需要将 5 向上对齐到 4 的倍数 → 对齐后为 8 字节
- 所以 `sizeof(U2)` = 8 字节

内存布局示意（假设地址从 0 开始）：

- 联合体起始地址必须是 4 的倍数（因为对齐要求 4）
- 分配 8 字节空间，虽然实际只用前 5 字节，但多余的 3 字节作为填充，保证下一个对象也能对齐到 4。

# 5.空类能实例化吗？

空类**可以**实例化。尽管空类没有任何成员变量或成员函数，但它仍然可以实例化。

每个实例会占用最小的内存（通常是 1 字节），以确保每个对象有一个唯一的地址。

**当一个类中没有任何成员变量和成员函数，这时**

**sizeof(A)的 A 值是多少？**

**空类**，即使一个类中没有任何成员变量和成员函数，其 sizeof（）值也为 1。

**这是因为编译器必须为每个对象分配一个独特的地址，以便在程序中区分不同的对象实例，**

# 8. C++类构造函数初始化列表执行顺序

**执行顺序**

- **基类构造函数**：
  -  首先调用基类的构造函数（如果存在）。
  - 如果有多个基类，则按照它们在**继承列表中的顺序**依次调用。

- **成员变量初始化**
  - ==成员变量的初始化顺序与在初始化列表中的顺序无关，只是与变量在类中的声明顺序有关==
  - 按照成员变量在类中声明的顺序进行初始化，而不是按照它们在初始化列表中的顺序。
  - 这种顺序确保所有依赖关系都能正确处理，避免未初始化的成员被使用。

- **构造函数主体**：
  - 最后执行**派生类构造函数**的主体部分

````c++
#include <iostream>

class Base{
public:
    Base(){
        std::cout<<"Base constructor called"<<std::endl;
    }
};

class Member2{
public:
    Member2(){
        std::cout<<"Memeber2 constructor called"<<std::endl;
    }
};
class Member1{
public:
    Member1(){
        std::cout<<"Memeber1 constructor called"<<std::endl;
    }
};
class Derived:public Base{
private:
    Member1 b1;
    Member2 b2;
public:
    Derived():b2(),b1(){
        std::cout<<"Derived constructor called"<<std::endl;
    }
};

int main()
{
    Derived d;
    return 0;
}
````

```
Base constructor called
Memeber1 constructor called
Memeber2 constructor called
Derived constructor called
```

- 基类构造函数:

  在执行子类Drived的构造函数之前,先调用基类`Base`的构造函数

- 成员变量的初始化

  成员变量的初始化顺序与它们在初始化列表中的顺序无关,只与它们在类中的声明顺序有关,因此先调用`Merber1`的构造函数进行初始化,再调用`Merber2`的构造函数进行初始化

- 构造函数主体

  最后执行子类`Derived`的构造函数主体部分

# 9.构造函数的顺序，析构函数的顺序

- **构造函数的顺序**

  - **基类构造函数**：首先调用**基类的构造函数**。

  -  **成员变量初始化**：按照**成员变量在类中声明的顺序进行初始化**。

  - **构造函数主体**：最后执行**派生类构造函数的主体部分**。

- **析构函数的顺序**

  - **析构函数主体**：首先执行**派生类析构函数的主体部分**。

  - **成员变量销毁**：按**成员变量在类中声明的顺序的反向销毁成员变量**。

  - **基类析构函数**：最后调用**基类的析构函数**。

````c++
#include <iostream>

class Base{
public:
    Base(){
        std::cout<<"Base constructor called"<<std::endl;
    }
    ~Base(){
        std::cout<<"基类的析构函数"<<std::endl;
    }
};

class Member2{
public:
    Member2(){
        std::cout<<"Memeber2 constructor called"<<std::endl;
    }
    ~Member2()
    {
        std::cout<<"Member2的析构函数"<<std::endl;
    }
};
class Member1{
public:
    Member1(){
        std::cout<<"Memeber1 constructor called"<<std::endl;
    }
    ~Member1()
    {
        std::cout<<"Member1的析构函数"<<std::endl;
    }
};
class Derived:public Base{
private:
    Member1 b1;
    Member2 b2;
public:
    Derived():b2(),b1(){
        std::cout<<"Derived constructor called"<<std::endl;
    }
    ~Derived()
    {
        std::cout<<"子类的析构函数"<<std::endl;
    }
};

int main()
{
    Derived d;
    return 0;
}
````

````c++

Member2的析构函数
Member1的析构函数
基类的析构函数
````

# 10. c++有哪些构造函数

| 构造函数类型       | 语法特征                         | 主要用途                                                     |
| :----------------- | :------------------------------- | :----------------------------------------------------------- |
| 1. 默认构造函数    | `MyClass();`                     | **无参**创建对象                                             |
| 2.带参数构造函数   | `MyClass(int, double);`          | **接受参数**的构造函数                                       |
| 3.拷贝构造函数     | `MyClass(const MyClass&);`       | 使用**已有对象**初始化新对象                                 |
| 4.**移动构造**函数 | `MyClass(MyClass&&);`            | 从**临时对象**初始化新对象(转移资源)                         |
| 5.转换构造函数     | 单参数构造函数（非explicit）     | 隐式类型转换                                                 |
| 6.委托构造函数     | 在初始化列表中调用另一个构造函数 | 代码复用，避免重复初始化逻辑                                 |
| 7.显式默认/删除    | `=default` 或 `=delete`          | 控制编译器生成的版本<br />防止**隐式类型转换**和**复制初始化** |

## 3. 拷贝构造函数

```c++
#include <iostream>

class MyClass {
public:
    MyClass() { std::cout << "默认构造函数\n"; }
    MyClass(int a,int data=0):m_a(a),m_data(new int(data)) { std::cout << "带参构造函数\n"; }
    MyClass(const MyClass& other):m_a(other.m_a) {   // 深拷贝示例
        m_data = new int(*other.m_data);
        std::cout << "拷贝构造函数\n";
    }
private:
    int* m_data;
    int m_a;
};
int main()
{

    MyClass a;//默认构造函数
    MyClass b(10,8);//带参构造函数
    MyClass c = b;      // 调用拷贝构造函数
}
```

## 4. 移动构造函数（C++11）

- 用右值引用（`MyClass&&`）参数，从临时对象“窃取”资源，避免深拷贝。
- 常用于资源管理类（如智能指针、容器）。

```c++
#include <iostream>

class MyClass {
public:
    MyClass() { std::cout << "默认构造函数\n"; }
    MyClass(int a,int data=0):m_a(a),m_data(new int(data)) {
        std::cout << "带参构造函数\n"; 
    }
    MyClass(const MyClass& other):m_a(other.m_a) {   // 深拷贝示例
        m_data = new int(*other.m_data);
        std::cout << "拷贝构造函数\n";
    }
    //移动构造函数
    MyClass(MyClass&& other) noexcept
        :m_a(other.m_a),m_data(other.m_data){
        other.m_data=nullptr;//转移资源,防止原对象析构时释放
        std::cout<<"移动构造函数\n";
    }
    ~MyClass(){
        std::cout<<"析构函数\n";
    }
private:
    int* m_data;
    int m_a;
};
int main()
{

    MyClass a;          //默认构造函数
    MyClass b(10,8);    //带参构造函数
    MyClass c = b;      // 调用拷贝构造函数

    MyClass d = std::move(c);  // 移动构造（c 被移走，之后不应再使用其资源）
    MyClass e = MyClass(20,5); 
    
    return 0;
}
```

```tex
1. 默认构造函数      → 对应 MyClass a;
2. 带参构造函数      → 对应 MyClass b(10,8);
3. 拷贝构造函数      → 对应 MyClass c = b;
4. 移动构造函数      → 对应 MyClass d = std::move(c);
5. 带参构造函数      → 对应 MyClass e = MyClass(20,5);  // 此处发生了 C++17 强制复制省略
6. 析构函数          → 对应 e 对象
7. 析构函数          → 对应 d 对象
8. 析构函数          → 对应 c 对象
9. 析构函数          → 对应 b 对象
10. 析构函数         → 对应 a 对象
```

- **`MyClass e = MyClass(20,5);`**
  - 这里 `MyClass(20,5)` 是一个临时对象（纯右值）。
  - 在 C++17 之前，会先构造临时对象`MyClass(20,5)`，再调用移动构造函数将临时对象的内容转移给 `e`，然后析构临时对象。
  - **C++17 起强制复制省略（copy elision）**：编译器直接构造 `e`，如同 `MyClass e(20,5);`。省略了临时对象的创建和移动/拷贝构造。
  - 因此，这里直接调用带参构造函数构造 `e`，输出 `带参构造函数`，而**没有**额外的移动构造函数和临时对象析构。

# 12. 全局变量和局部变量的区别

```c++
#include <iostream>
using namespace std;

// 全局变量：在函数外定义，整个程序生命周期有效
int globalVar = 100;

void func() {
    // 局部变量：在函数内定义，仅在该函数内部可见
    int localVar = 200;
    cout << "在 func() 中：globalVar = " << globalVar << ", localVar = " << localVar << endl;
    
    // 局部变量可以覆盖同名全局变量（但通常不推荐）
    // 如果定义同名局部变量，会隐藏全局变量
    int globalVar = 999;  // 局部变量，与全局变量同名
    cout << "在 func() 中（局部覆盖后）：globalVar = " << globalVar << endl;
    // 如果想访问被覆盖的全局变量，可以使用作用域解析符 ::
    cout << "在 func() 中,显式地访问: globalVar = " << ::globalVar << endl;
}

int main() {
    // 局部变量 main 函数内
    int localVar = 300;
    cout << "在 main() 中：globalVar = " << globalVar << ", localVar = " << localVar << endl;
    
    func();  // 调用函数，观察其内部的局部变量
    
    // 函数调用结束后，func 内的局部变量 localVar 和覆盖的 globalVar 均被销毁
    cout << "返回 main() 后，globalVar = " << globalVar << ", localVar = " << localVar << endl;
    
    return 0;
}
```

```tex
在 main() 中：globalVar = 100, localVar = 300
在 func() 中：globalVar = 100, localVar = 200
在 func() 中（局部覆盖后）：globalVar = 999
在 func() 中,显式地访问: globalVar = 100
返回 main() 后，globalVar = 100, localVar = 300
```

- **全局变量**：在函数外定义，默认初始化为 0（数值类型），生命周期贯穿整个程序运行。
- **局部变量**：在函数或代码块内定义，未显式初始化时内容不确定（建议总是初始化），生命周期在函数返回或块结束时结束。
- **作用域**：局部变量优先于同名的全局变量（局部隐藏全局），但可通过 `::` 访问被隐藏的全局变量。

# 15.shared_ptr 是不是线程安全的？

`shared_ptr` 的**引用计数操作是线程安全**的，确保引用计数的增减不会引发竞争条件。

然而，`shared_ptr` 所**管理的对象本身并不是线程安全的**，**读写操作需要额外的同步机制**来

保证线程安全。

==参看:== [12全局算法.md thread](..\c++_md\3高级面向对象\12全局算法\12全局算法.md) 

```c++
#include <iostream>
#include <memory>
#include <thread>
#include <vector>

void threadFunction(std::shared_ptr<int> ptr){
    std::cout<<"Thread:"<<std::this_thread::get_id()<<",value:"<<*ptr<<std::endl;
}

int main(){
    std::shared_ptr<int> sharedptr = std::make_shared<int>(100);
    std::vector<std::thread> threads;

    for(int i=0;i<10;i++){
        threads.push_back(std::thread(threadFunction,sharedptr));
    }

    for(auto& th:threads){
        th.join();
    }

    return 0;
}
```

# 17. shared_ptr和weak_ptr的区别

- **所有权**:shared_ptr拥有资源的**共享所有权**;	weak_ptr不拥有资源的所有权,它是一个弱指针
- **引用计数**:shared_ptr增加引用计数;                           weak_ptr不增加引用计数
- **生命周期控制**:shared_ptr控制资源的生命周期;        weak_ptr不控制资源的生命周期
- **循环引用**:shared_ptr **可能导致循环引用,内存泄漏**;   weak_ptr可以**打破循环引用,防止内存泄漏**

`shared_ptr` 拥有资源的共享所有权并控制资源的生命周期，增加引用计数，而`weak_ptr` 仅提供对资源的弱引用，不控制资源的生命周期，也不增加引用计数。`weak_ptr`常用于打破循环引用，防止内存泄漏

## 循环引用

```c++
#include <iostream>
#include <memory>

class Node{
public:
    std::shared_ptr<Node> next;
    ~Node(){std::cout<<"Node destroyed"<<std::endl;}
};

void createCyle(){
    //等价于:
        // std::shared_ptr node1(new Node());
        // std::shared_ptr node1 = std::make_shared<Node>()
    auto node1 = std::make_shared<Node>();//动态分配一个Node类型的对象,并返回一个管理该对象的shared_ptr
    auto node2 = std::make_shared<Node>();;
    node1->next = node2;
    node2->next = node1;//循环引用
}

int main()
{
    createCyle();
    return 0;
}
```

1. 在 `createCyle` 中创建两个 `shared_ptr<Node>`：
   - `node1` 指向一个 `Node` 对象（称对象1,Node1），引用计数为 1。①
   - `node2` 指向另一个 `Node` 对象（称对象2,Node2），引用计数为 1。②
2. 执行 `node1->next = node2`：
   - 对象1的成员 `next` 指向对象2，对象2的引用计数增加为 2。②
3. 执行 `node2->next = node1`：
   - 对象2的成员 `next` 指向对象1，对象1的引用计数增加为 2。①

4. 当函数`createCyle`结束时

- 局部变量`node1`销毁 → Node1的引用计数 从 2减为1(来自Node2的next)
- 局部变量`node2`销毁 → Node2的引用计数 从 2减为1(来自Node1的next)
- 此时,Node1对象仍然被Node2对象的next引用,Node2对象仍然被Node1对象的next引用,形成循环引用。两个对象的引用计数均为 1，不为零，因此它们永远不会被销毁，内存泄漏发生。
  - 只有当 shared_ptr管理的对象的引用计数为0时,才会调用当前对象的析构函数,否则不销毁当前对象。

## weak_ptr解决

![weakptr解决](D:\Typora\typora_work\c++_md\3高级面向对象\12全局算法\weakptr解决.png)

```c++
#include <iostream>
#include <memory>

class Node{
public:
    std::shared_ptr<Node> next;
    std::weak_ptr<Node> prev;  
    ~Node(){std::cout<<"Node destroyed"<<std::endl;}
};

void createCycleWithWeakPtr() {
    auto node1 = std::make_shared<Node>();
    auto node2 = std::make_shared<Node>();
    node1->next = node2;
    node2->prev = node1; // 打破循环引用
}

int main()
{
    createCycleWithWeakPtr();
    return 0;
}

//Node destroyed
//Node destroyed  
```

- 由于	`weak_ptr` 不增加引用计数，因此 `node2->prev = node1` 不会增加 `node1` 的引用计数。
- 当函数`createCycleWithWeakPtr`结束时,
- 
- 局部变量`node2`销毁 → Node2对象的引用计数减为1(来自Node1的next)
- 局部变量`node1`销毁 → Node1对象的引用计数归0 → 调用Node1的析构函数
- Node1的析构函数导致 next成员销毁 →  Node2对象的引用计数归0 → 调用Node2的析构函数

# 18. 引用作为函数参数在 C++中有以下几个好处？

在 C++ 中，将引用作为函数参数有以下几个主要好处：

1. **避免拷贝，支持大对象传递提高效率**
   对于大型对象（如 `std::string`、`std::vector` 或自定义类），传值会触发拷贝构造函数，开销较大。引用传递**直接操作原对象，无需拷贝，显著提升性能。**

2. **允许修改实参**
   通过引用传递，**函数内对参数的修改会直接影响外部实参，类似指针的效果，但语法更简洁**。例如：

   ```cpp
   void increment(int &x) { x++; }  // 直接修改传入的变量
   ```

3. **避免空指针问题**
   引用必须在定义时绑定到有效对象，且不能为 `nullptr`（除非通过不安全的强制转换）。相比指针，使用引用作为参数无需检查空值，代码更安全、更清晰。

4. **语法自然，易于阅读**
   引用传递的调用语法与传值完全相同（直接写变量名），无需取地址 `&` 或解引用 `*`，降低了使用者的心智负担。

5. **支持运算符重载与拷贝控制**
   许多运算符重载（如赋值 `=`、下标 `[]`、输入输出 `<<` / `>>`）必须或更适合使用引用参数，以实现链式操作或直接修改对象。

6. **常量引用额外扩展了功能**
   `const` 引用可以绑定到临时对象（右值）和字面量，允许传递常量或表达式结果，同时仍避免拷贝。例如：

   ```cpp
   void print(const std::string &s);  // 可接受 "hello" 或 string("abc")
   ```

**总结**：引用参数结合了指针的高效与值参数的简洁，是现代 C++ 中推荐传递非内置类型参数的方式（尤其是配合 `const` 只读访问时）。

````c++
#include <iostream>
#include <vector>

//避免拷贝开销和支持大对象传递
void processLargeVector(const std::vector<int>& vec){
    std::cout<<"Vector size:"<<vec.size()<<std::endl;
}

//支持修改实参
void updateVale(int &x){
    x = 42;
}

//一致性和安全性
void printValue(const int& x){
    std::cout<<"Value:"<<x<<std::endl;
}

// 常引用示例 绑定到左值或右值
void printString(const std::string& str){
    std::cout << "String content: " << str << std::endl;
}

int main(){
    int a = 10;
    std::cout << "Original value: " << a << std::endl;
    updateVale(a);
    std::cout << "Updated value: " << a << std::endl;
    printValue(a); //传递左值

    // 常引用的好处：可以传递临时对象（右值）或字面量
    printValue(100);        // 直接传递 int 字面量，常引用绑定到临时 int
    printString("Hello, constant reference!");  // 传递字符串字面量，常引用绑定到临时 std::string
    
    std::vector<int> largeVector(10000,1);
    processLargeVector(largeVector);    // 使用引用避免拷贝大对象
    std::string largeString(10000,'a');
    printValue(largeString.length());   // 打印大字符串的长度
    
}
````

# 19. 模版在什么时候实例化

​	**模版在编译时实例化**,当编译器遇到模版的具体使用时,如**调用模版函数或者创建模版类实例**时,编译器会生成响应的代码。这一过程称为**模版的实例化**

​	模板本身只是一个“蓝图”或“公式”，编译器在**编译阶段**（确切地说是编译期，部分实例化信息也可能保留到链接期）遇到模板的**具体使用**时（比如**调用函数模板**、创建模版类实例(定义类模板对象)、取模板函数地址等），编译器才会根据模板参数生成真正的函数或类代码。这个过程就叫**模板的实例化**。

```c++
#include <iostream>

template <typename T>
class MyClass {
public:
    T data;
    MyClass(T d) : data(d) {}//类的构造函数模版
    void display() const {
        std::cout << "Data: " << data << std::endl;
    }
};

// ---------- 显式实例化（通常放在 .cpp 文件中）----------
// 显式实例化 MyClass<int> 的全部成员
template class MyClass<int>;

// 显式实例化 MyClass<double> 的 display 成员函数（其余成员按需实例化）
template void MyClass<double>::display() const;
// -----------------------------------------------------

int main() {
    MyClass<int> obj1(42);    // 使用已显式实例化的 int 版本，不会重新生成代码
    MyClass<double> obj2(3.14); // double 版本中只有 display 被显式实例化，
                                 // 构造函数等仍会隐式实例化（因为需要定义对象）

    obj1.display();
    obj2.display();
    return 0;
}

```

# 20. 为什么模版代码一般都是放在一个.h头文件中

### 1. **普通函数的声明与定义分离（标准做法）**

头文件负责**“教编译器怎么做”**（提供源码），`.cpp` 文件负责**“告诉编译器做什么”**（调用代码）。



在普通函数的声明与定义在C++项目中通常是分离的,这是 C++ 项目中最基础的组织方式，目的是隐藏实现细节并加快编译速度。

**第一步：头文件 (**`math_utils.h`**)**

只写**声明**（告诉编译器有这么个函数，长什么样）。

```c++
// math_utils.h
#ifndef MATH_UTILS_H
#define MATH_UTILS_H

// 函数声明：只给原型，没有函数体
int add(int a, int b);

#endif
```

**第二步：源文件 (**`math_utils.cpp`**)**

写**定义**（具体的实现逻辑）。

```c++
// math_utils.cpp
#include "math_utils.h"

// 函数定义：具体的实现
int add(int a, int b) {
    return a + b;
}
```

**第三步：主程序 (**`main.cpp`**)**

只包含头文件即可使用。

```c++
// main.cpp
#include <iostream>
#include "math_utils.h" // 只需要包含头文件

int main() {
    int result = add(10, 20); // 编译器知道 add 存在，链接器会在 .cpp 中找到它
    std::cout << "Result: " << result << std::endl;
    return 0;
}
```

**结果**：

```bash
g++ main.cpp math_utils.cpp -o my_program
```

编译链接成功。因为 `main.cpp` 编译时只需要知道 `add` 的声明，链接时才去 `math_utils.cpp` 生成的 `.o` 文件里找具体的代码。

### 2. **模板的声明与定义分离（特殊情况）**

将 模版代码一般都是放在一个.h头文件中 **这句话的意思是**:

1. **模版的声明和定义必须在一起**（都在头文件里）。
2. **使用代码必须在别处**（在 `.cpp` 文件里）。



模板的**实例化**需要看到**完整定义**。==通常将模板的声明和定义都放在头文件中==，否则会出现**链接错误**（因为其他 .cpp 看不到定义，无法实例化）

**编译时实例化** 

​	模板的实例化发生在编译时，编译器需要看到模板的定义和实现

**链接问题** 

​	如果模板定义和实现分开，编译器可能无法在链接时找到实现

**总结：**

**(1)模板类在编译期实例化：**模板类的实例化发生在编译期，编译器需要在实例化

模板时看到模板的完整定义和实现。

**(2)头文件包含完整定义和实现：**将模板类的定义和实现都放在头文件中，确保编

译器在实例化模板时能访问到完整信息，避免链接错误。

**(3)避免链接错误：**如果将模板类的实现放在源文件中，可能会导致链接错误，因为

编译器在链接阶段无法找到模板实例化的具体实现。

C++ 模板（包括模板类和模板函数）通常将定义全部放在头文件中，这主要源于 **编译模型** 的限制和模板实例化的特点。具体原因如下：

## 1. 模板实例化需要完整定义
模板是一个“代码生成器”，编译器在**编译期**看到具体使用（如 `MyClass<int>`）时，必须知道模板的完整定义（包括成员函数的函数体），才能生成对应的机器码。  
如果将模板的声明放在头文件、定义放在 `.cpp` 文件，那么在使用模板的 `.cpp` 文件中只能看到声明，看不到定义，编译器就无法实例化，从而导致**链接错误**（未解析的外部符号）。

## 2. 传统 C++ 不支持“分离编译模式”
C++ 标准没有提供类似 `export`（C++11 已废弃）的机制来支持模板的分离编译。虽然 C++20 引入了模块（module），但主流编译器对模板的传统处理仍以头文件为主。

## 3. 头文件方式简单可靠
将所有模板代码放入头文件（`.h` 或 `.hpp`），然后在每个使用该模板的 `.cpp` 文件中包含该头文件。每个翻译单元（`.cpp`）会各自实例化自己用到的模板实例，链接器会合并重复的实例化代码。这种方式被广泛采用，且没有额外的编译复杂度和性能损失。

## 4. 显式实例化是一种替代方案，但不常用
如果您真的希望将模板定义分离到 `.cpp` 文件中，可以：
- 在 `.cpp` 文件中**显式实例化**所有可能用到的模板类型（如 `template class MyClass<int>;`）。
- 然后在头文件中只保留声明。

但这种做法缺点明显：
- 需要提前知道所有可能的模板参数类型，缺乏灵活性。
- 对于库作者来说，无法让用户使用自定义类型实例化模板。

## 5. 现代 C++ 的改进：模块（C++20）
C++20 引入了模块，可以更优雅地解决模板的编译隔离问题，不再强制要求将模板实现放在头文件中。但目前（2026 年）许多项目仍在迁移阶段，传统头文件方式依然是最通用、最兼容的做法。

## 总结
> **模板类放在头文件中是为了让编译器在使用模板时能看到完整定义，从而能够实例化生成代码。** 这是 C++ 模板编译模型的直接结果，也是工程实践中最安全、最灵活的做法。

# 21. define宏定义和const,static的区别

`define`宏定义通过预处理器在编译时简单的进行文本替换,缺乏类型检查,适用于定义简单的常量或宏。

`const`用于定义常量，能够进行类型检查且必须进行初始化，不能在声明后修改；具有更高的安全性。

`static`用于定义静态变量或成员，具有文件内或函数内的局部作用域，并且在程序的生命周期内保持值的持续性。

## 当`static`作用于全局变量/函数上

-  表示示该变量/函数具有**内部链接**，仅在当前文件内可见，其他文件无法通过 `extern` 访问。
- 存储期为静态存储期（程序运行期间一直存在）。

## 当`static`作用于局部变量上

- 表示变量具有**静态存储期**，但作用域仍是块作用域。

- 变量在函数内保持其值跨调用，只初始化一次，。

### **跨调用保持**：	

- 是指变量在函数多次调用之间能够保留其值，不会因为函数返回而被销毁或重置。

```c++
void func() {
    static int count = 0;   // 局部静态变量
    int temp = 0;           // 局部自动变量

    count++;
    temp++;
    std::cout << count << " " << temp << std::endl;
}

int main() {
    func();   // 输出：1 1
    func();   // 输出：2 1
    func();   // 输出：3 1
}
```

- `count` 是局部静态变量，具有**静态存储期**。它在第一次调用时初始化，之后每次调用都会使用同一个变量，值不断累加——这就是**跨调用保持**。
- `temp` 是局部自动变量，具有**自动存储期**。每次调用都会重新创建并初始化为 `0`，函数返回时销毁，所以它的值不会跨调用保持。

#### 常见应用场景

1. **计数器**（如上面的例子）
2. **单例模式**（在函数内返回静态局部实例）
3. **缓存**（在函数内缓存计算结果，避免重复计算）
4. **状态机**（记录函数调用间的状态）

## 当`static`作用于类成员上，

- `static`成员变量表示当前成员变量属于整个类本身而非实例，所有对象共享一份。

- `static`成员函数没有`this`指针，只能访问静态成员。

## 表格对比

| 特性           | `#define`                     | `const`                                        | `static`                                      |
| :------------- | :---------------------------- | :--------------------------------------------- | :-------------------------------------------- |
| **处理阶段**   | 预处理（文本替换）            | 编译期                                         | 编译期                                        |
| **类型检查**   | 无                            | 有                                             | 有（变量类型）                                |
| **存储期**     | 无                            | 自动或静态（依位置）                           | 静态（全局/局部静态）/ 类作用域               |
| **作用域**     | 文件（从定义到结束/`#undef`） | 普通作用域                                     | 文件/块/类                                    |
| **链接性**     | 不适用                        | 内部（全局`const`）或外部（局部`const`无链接） | 内部（全局`static`）或无链接（局部`static`）  |
| **是否占内存** | 否（宏展开为字面量）          | 是（通常，但可能被优化）                       | 是                                            |
| **能否修改**   | 无变量，不存在修改            | 不可修改                                       | 可修改                                        |
| **初始化**     | 不适用（只是替换）            | 必须初始化                                     | 默认初始化为0，可显式初始化                   |
| **典型用途**   | 常量、条件编译、宏函数        | 类型安全常量、函数参数保护                     | 文件局部变量/函数、持久化局部状态、类共享数据 |

- **用 `const` 代替 `#define` 定义常量**（类型安全，便于调试）。
- **用 `static` 控制变量/函数的作用域**（隐藏实现细节）。
- **用 `#define`** 的场景：
  - 条件编译（`#ifdef`）
  - 跨平台适配代码
  - 必须进行文本替换的宏（如日志宏、调试宏）
  - 创建重复的代码片段（但要小心副作用）
