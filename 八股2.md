## 深度递归导致栈溢出

**深度递归导致栈溢出**，指的是当一个函数（或一组函数）递归调用的层数过深时，程序运行时使用的**调用栈（Call Stack）**空间被耗尽，从而引发崩溃（如 `Segmentation Fault` 或 `Stack Overflow`）。

下面用通俗的方式解释这个过程。

### 1. 什么是“调用栈”？

每次调用一个函数，系统都会在内存的**栈区**为该函数分配一块连续的空间，称为**栈帧（Stack Frame）**。栈帧里存放着：

-   函数的参数
-   返回地址（调用结束后该回到哪里）
-   局部变量
-   一些保存的寄存器

**栈的大小是有限的**（通常在 Windows/Linux 上默认为 **1~8 MB**）。当函数返回时，对应的栈帧会被销毁（栈顶指针回退）。

### 2. 递归与栈帧的累积

递归函数会**调用自身**。每调用一次，就会在栈上**压入一个新的栈帧**，**不会**因为函数体相同就复用旧帧。

例如，一个简单的递归求阶乘：

```cpp
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);   // 递归调用
}
```

调用 `factorial(100000)` 时，会发生：

1.  调用 `factorial(100000)` → 压入栈帧1（n=100000）
2.  内部调用 `factorial(99999)` → 压入栈帧2（n=99999）
3.  内部调用 `factorial(99998)` → 压入栈帧3（n=99998）
4.  ...
5.  一直持续到 `factorial(1)` 才会开始返回。

**问题**：在到达基准情况之前，栈上同时存在着 10 万个栈帧。每个栈帧即使很小（比如 32 字节），10 万个也会占用约 3.2 MB。但实际中栈帧可能更大（保存更多寄存器、对齐等），很容易超过系统栈限制（例如 1 MB），于是发生**栈溢出**。

### 3. 为什么循环不会栈溢出？

如果用循环实现同样功能：

```cpp
int factorial_loop(int n) {
    int result = 1;
    for (int i = 2; i <= n; ++i) result *= i;
    return result;
}
```

无论 `n` 多大，栈上**始终只有一个栈帧**（`factorial_loop` 自己的帧），不会累积，因此不会栈溢出（但 `result` 可能会溢出，那是另一个问题）。

### 4. 深度递归的典型后果

-   **编译/运行时报错**：常见错误如 `Segmentation fault`（Linux）或 `Stack overflow`（Windows / Rust / .NET）。
-   **程序直接崩溃**，没有优雅的恢复机制（除非使用特殊手段如 guard pages 或异常转换）。
-   **无法通过增加递归深度参数解决**，因为栈大小是有限的，极深的递归（如数十万层）几乎一定会超限。

### 5. 如何避免或缓解？

-   **改用迭代**：把递归算法手动改写为循环（最推荐）。
-   **尾递归优化（Tail Call Optimization, TCO）**：如果递归调用是函数的最后一个操作（尾递归），某些编译器（如 GCC、Clang 开启 `-O2`）可以复用当前栈帧，避免增长。但 C++ 标准不强制要求，且尾递归形式有局限。
-   **增大栈大小**：编译器链接选项（如 `-Wl,--stack,16777216` 在 Windows 上设 16 MB）或系统命令（`ulimit -s` 在 Linux 上），但治标不治本。
-   **改用堆模拟栈**：用 `std::stack` 在堆上显式管理状态，完全避免递归调用。

### 6. 一个直观的代码示例

```cpp
#include <iostream>

void recurse(int depth) {
    char buffer[1024];  // 故意让每个栈帧大一点（1KB）
    std::cout << "Depth: " << depth << std::endl;
    recurse(depth + 1);
}

int main() {
    recurse(1);  // 大约几百到几千次后就会崩溃
    return 0;
}
```

运行这个程序，很快会看到崩溃，因为每个栈帧至少 1KB，栈大小 8MB 时只能容纳约 8000 层。

### 总结

| 概念         | 解释                                                         |
| :----------- | :----------------------------------------------------------- |
| **深度递归** | 函数调用链的层数非常大（例如数万、数十万）。                 |
| **栈溢出**   | 程序向栈区申请的空间超过了操作系统为栈预留的总大小。         |
| **根本原因** | 每一层递归都要分配独立的栈帧，且只有等最内层返回后才能逐层释放；栈空间有限。 |
| **典型症状** | 程序崩溃，错误信息包含 `stack overflow` 或 `segmentation fault`。 |
| **解决方向** | 改用迭代、尾递归优化、增大栈（不推荐）、用堆模拟递归。       |

理解这个机制对于编写健壮的递归函数（如深度优先搜索、递归解析器）非常重要。

# 27. **介绍一下 RAII 思想，他怎么保证对象没有内存泄漏？**

 [第4章复合类型.md](..\C++ Primer Plus\第4章复合类型.md) 

**RAII**（Resource Acquisition Is Initialization）是 C++中的一种重要思想,确保资源在对象的生命周期内得到正确管理。**在对象创建时获取资源，在对象析构时释放资源**，避免了资源泄漏和未定义行为。通过 RAII，程序能够自动管理资源，即使在**异常发生时也能保证资源被正确释放**。

```c++
#include <iostream>
#include <fstream>

class FileRAII{
    std::fstream file;// 成员变量：一个文件流对象
public:
    FileRAII(const std::string& filename){
        file.open(filename,std::ios::out | std::ios::trunc);
        if(!file.is_open()){
            throw std::runtime_error("Unable to open file");
        }
    }
    ~FileRAII(){
        if(file.is_open()){
            file.close();
        }
    }
    void write(const std::string& content){
        if (file.is_open()) {
            file << content << std::endl;
        }
    }
};
int main() {
    try {
        FileRAII file("example.txt");
        file.write("Hello, RAII!");
    }
    catch (const std::exception& e) {
        std::cerr << "Exception: " << e.what() << std::endl;
    }
    return 0;
}
```

#include <iostream>

    virtual ~Base() {}
};

class Derived : public Base {
public:
    // 覆盖基类中的虚函数
    void show() override {
        std::cout << "Derived class show function called." << std::endl;
    }
};

void function(Base& b) {
    b.show(); // 在运行时确定调用哪个版本的 show()
}

**(1)构造函数不能是虚函数：**

---

## 1. 函数指针 —— 传统的回调式多态

函数指针允许将不同的函数地址赋给同一个指针变量，从而实现“同一个调用形式，不同行为”的运行时多态。

```cpp
#include <iostream>

void bark() { std::cout << "Woof!\n"; }
void meow() { std::cout << "Meow!\n"; }

int main() {
    void (*sound)();   // 函数指针类型
    sound = bark;
    sound();           // 输出 Woof!
    sound = meow;
    sound();           // 输出 Meow!
}
```

**多态体现**：同一个 `sound()` 调用，根据指针指向的函数不同，执行不同行为。

---

## 2. `std::function` —— 更通用的可调用对象包装器

`std::function` 可以存储任何可调用对象（函数、lambda、函数对象等），实现类型擦除的运行时多态。

```cpp
#include <iostream>
#include <functional>

void bark() { std::cout << "Woof!\n"; }
struct Meow { void operator()() const { std::cout << "Meow!\n"; } };
//operator重载()运算符,参数为()空

int main() {
    std::function<void()> sound;
    sound = bark;
    sound();               // Woof!
    sound = Meow();
    sound();               // Meow!
    sound = [](){ std::cout << "Quack!\n"; };
    sound();               // Quack!
}
```

**多态体现**：`sound()` 可以代表多种不同的可调用实体，调用时执行各自的行为。

## 3. 手动类型擦除 —— 模拟虚函数机制

类型擦除的核心思想：定义一个非模板基类（纯接口），然后模板派生类包装具体类型，通过基类指针调用虚函数（但这里我们不用语言自带的虚函数，而是用函数指针手动模拟）。实际上，手动类型擦除通常仍然需要虚函数或函数指针，但我们可以用**函数指针成员**来代替虚表。

```cpp
#include <iostream>
#include <memory>

// 类型擦除的基类（非模板）
class AnimalConcept {
public:
    void speak() { speak_func(this); }
protected:
    using SpeakFunc = void(*)(AnimalConcept*);
    AnimalConcept(SpeakFunc sf) : speak_func(sf) {}
private:
    SpeakFunc speak_func;
};

// 模板派生类：存储具体类型 T，并提供静态函数来调用 T 的 speak
template<typename T>
class AnimalModel : public AnimalConcept {
public:
    AnimalModel(T t) : AnimalConcept(&AnimalModel::speak_impl), data(std::move(t)) {}
private:
    T data;
    static void speak_impl(AnimalConcept* self) {
        auto* model = static_cast<AnimalModel<T>*>(self);
        model->data.speak();
    }
};

// 具体类型
struct Dog { void speak() const { std::cout << "Woof!\n"; } };
struct Cat { void speak() const { std::cout << "Meow!\n"; } };

int main() {
    AnimalConcept* p = new AnimalModel<Dog>(Dog{});
    p->speak();   // Woof!
    delete p;

    AnimalConcept* p2 = new AnimalModel<Cat>(Cat{});
    p2->speak();  // Meow!
    delete p2;
}
```

**多态体现**：通过基类指针 `AnimalConcept*` 调用 `speak()`，实际执行的是具体类型（`Dog` 或 `Cat`）的 `speak` 方法。这里我们没有使用 C++ 的 `virtual` 关键字，而是手动用函数指针模拟了虚表。

## 4. 模板 —— 编译时多态

模板在编译期根据实际类型生成不同代码，实现静态多态（也称为“无界多态”）。

```cpp
#include <iostream>

struct Dog { void speak() const { std::cout << "Woof!\n"; } };
struct Cat { void speak() const { std::cout << "Meow!\n"; } };

template<typename T>
void makeSound(const T& animal) {
    animal.speak();   // 编译时绑定，要求 T 有 speak 方法
}

int main() {
    makeSound(Dog{});   // 实例化 makeSound<Dog>
    makeSound(Cat{});   // 实例化 makeSound<Cat>
}
```

**多态体现**：同一段模板代码 `makeSound` 可以接受不同的类型，产生不同的行为。这是编译时多态，没有运行时开销。

## 总结对比

| 方法               | 多态类型   | 运行时开销                     | 适用场景                            |
| :----------------- | :--------- | :----------------------------- | :---------------------------------- |
| 虚函数（语言内置） | 运行时动态 | 一次间接寻址                   | 面向对象设计，需要灵活扩展          |
| 函数指针           | 运行时动态 | 一次间接寻址                   | C 风格回调，简单场景                |
| `std::function`    | 运行时动态 | 稍高（类型擦除+可能的堆分配）  | 需要存储任意可调用对象              |
| 手动类型擦除       | 运行时动态 | 类似虚函数（自己维护函数指针） | 需要定制虚表或避免 `virtual` 关键字 |
| 模板               | 编译时静态 | 零（内联）                     | 泛型编程，性能敏感，类型已知        |

可以看到，C++ 实现多态的方式非常灵活：虚函数只是其中一种，且是专门为运行时多态设计的语言特性。其他方法可以在特定场景下替代或补充虚函数。

# 35. 同一个类的不同对象的虚函数表是同一个吗？

是的，**同一个类的所有对象共享同一个虚函数表**。每个对象中都有一个虚指针（vptr），

指向该类的虚函数表。因此，同一个类的不同对象的虚函数调用会通过相同的虚函数表进行。

# 36. 基类的虚函数表存放在内存的什么区，虚表指针vptr 的初始化时间？

​	**表. 虚函数表的特征和存储位置表**

| 特征         | 详细描述                                                     |
| ------------ | ------------------------------------------------------------ |
| **全局共享** | 虚函数表是全局共享的，每个类型只有一个虚函数表。在编译时就已经构造完成。 |
| **存储内容** | 虚函数表存储的是虚函数的地址，它类似于一个数组。每个类对象中都存储有一个指向其虚函数表的指针（vptr）。 |
| **类比**     | 由于虚函数表的特性类似于类中的静态成员变量，它们都是全局共享且大小固定。 |
| **动态分配** | 虚函数表的大小在编译时已确定，无需在运行时动态分配内存，因此不存储在堆区。 |
| **内存区域** | 在 Linux/Unix 系统中，虚函数表通常存放在可执行文件的**只读数据段**（.rodata） |

​	**表. 虚表指针 (vptr) 的初始化时间和存储位置**

| 特征           | 详细描述                                                     |
| -------------- | ------------------------------------------------------------ |
| **初始化时机** | 虚表指针 vptr 在类的构造函数执行时进行初始化。它是类与虚函数密不可分的一部分。 |
| **存储位置**   | vptr 存在于对象内存布局的最前面，它指向对应的虚函数表。      |
| **内存区域**   | 虚表指针作为对象的一部分，存储在对象所在的内存区域，例如如果对象是局部变量则在栈上，动态分配则在堆上。 |

## 总结

- ​	**虚函数表（vtable）：** 虚函数表（vtable）是实现多态性的关键组件。**每个包含虚函数**

**的类都有一个全局共享的虚函数表，它在编译时构建完成并存储在程序的只读数据段**

**（.rodata）中。**虚函数表类似于一个指针数组，其中每个元素指向一个具体的虚函数实现。

这种机制使得程序在运行时能够根据对象的实际类型调用相应的虚函数，从而实现动态多态。

- ​	**虚表指针（vptr）：**虚表指针（vptr）则是类实例中的一个指针，**位于对象内存布局的**

**最前端**，**指向相应的虚函数表。**虚表指针的初始化发生在类构造函数的执行过程中，确保每

个对象实例都正确地链接到其类型特定的虚函数表。由于虚表指针的存在，即使使用基类的

指针或引用来操作对象，程序也能通过 vptr 找到正确的虚函数表，并调用到派生类中的相

应虚函数，实现了类型的多态性行为。

# 37. -35 虚函数表，是对象拥有还是类拥有？

**虚函数表（vtable）是类拥有的，**而不是对象拥有的。每个包含虚函数的类都有一个虚函数表，表中存储了指向该类虚函数实现的指针。对象中有一个指向该虚函数表的指针（通常称为虚指针，**vptr**）。

# 38. 虚函数内部调用非虚函数是调用指针类还是对象类?

当一个虚函数在调用时，调用的是**实际对象**的虚函数实现，这是通过虚函数表（vtable）

来实现的。然而，非虚函数的调用是静态绑定的，即在编译时就已经确定了调用哪个函数。

因此，当虚函数内部调用非虚函数时，调用的是当前对象所属类的非虚函数。

```c++
#include <iostream>
class Base {
public:
    virtual void show() {
        print(); // 调用的是 Derived 类的 print
    }
    void print() {
        std::cout << "Base print" << std::endl;
    }
};

class Derived : public Base {
public:
    void show() override {
        print(); // 调用的是 Derived 类的 print
    }
    void print() {
        std::cout << "Derived print" << std::endl;
    }
};
int main() {
    Derived d;
    Base* b = &d;
    b->show(); // 调用 Derived 的 show，继而调用 Derived 的 print
    return 0;
}
```

**调用路径**：

​	在 main 函数中，通过 Base 类型的指针 b 调用了 show 函数。

​	由于 show 是虚函数，调用的是实际对象 d 的 Derived 类中的 show 实现。

**内部调用**：

​	在 Derived 类的 show 函数中调用了 print 函数。

​	print 不是虚函数，调用的是 Derived 类中的 print 函数。

# ==c++什么变量存储在静态存储区?==

”**C++ 中什么变量存储在静态存储区？**“（下文详细回答部分提供了底层原理图示）

 [第4章复合类型.md](..\C++ Primer Plus\第4章复合类型.md) 

[数据结构.md](..\2数据结构与基本算法\数据结构.md)

## **简要回答**

在 C++ 中，具有**静态存储期**的变量会存储在静态区，例如全局变量、命名空间作用域变量、static 修饰的全局变量、static 修饰的局部变量、static 类成员变量、字符串常量等。

## **详细回答**

静态存储区中的变量不随函数调用创建和销毁，生命周期贯穿整个程序运行期间，静态存储区通常分为三类：

**1. 已初始化数据段(.data)：**存储已初始化的全局变量和静态变量。

**2. 未初始化数据段(.bss)：**存储未初始化或零初始化的全局变量和静态变量。

**3. 只读数据段(.rodata)：**存储只读常量，如字符串常量。

C++ 中以下变量会存储在静态存储区：

1.全局变量

```c++
int a = 10; // 已初始化，存储在 .data 段
int b; // 未初始化，存储在 .bss 段
```

2.命名空间作用域变量

```c++
namespace MyNamespace {
    int x = 10;
}

namespace { // 匿名命名空间
    int y = 10;
}
```

3.static 全局变量

```c++
static int a = 0;
```

4.static 局部变量

```c++
void func() {
 static int a = 0;
}
```

5.static 类成员变量

```c++
class A {
public:
    static int num; // 静态类成员变量
};

int A::num = 5;
```

6.字符串常量

```c++
const char* p = "Hello World!"; // "Hello World!" 这个字符串常量存储在静态存储区 .rodata 段
```

## **知识拓展**

面试官可能追问：

Q1： .data 段 和 .bss 段有什么区别？

​	A1：.data 段存放已初始化的全局变量和静态变量，.bss 段存放未初始化的全局变量和静态变量，或者初始化为 0 的全局变量和静态变量。

Q2：static 局部变量为什么不能存储在栈上？

​	A2：因为 static 局部变量的生命周期贯穿整个程序，而栈上的变量函数调用时创建，函数返回时销毁，如果存储在栈上，函数返回后就会被销毁，无法满足静态变量生命周期要求。

Q3：全局 const 变量存储在静态存储区吗？

​	A3：是的，全局 const 变量通常存储在静态区的 .rodata 段，不过有时候会被编译器优化掉，视为编译期常量，直接嵌入到使用它的代码中，从而不会分配内存空间。
